# Project Memory

This file serves as a shared knowledge base for all agents working on this project.
Record important architectural decisions, project conventions, environment setups, and persistent context here.

## App Overview
- **App Name**: Time to Go
- **Purpose**: User sets a home address (with autocomplete) and configures a daily alarm. When the alarm fires, the app fetches the user's GPS location, queries for public transit (bus-preferred) directions to home, and sends a local push notification with the route summary. Tapping the notification always opens Google Maps with transit directions.

## Technical Stack
- **Language**: Kotlin only. No Java source files.
- **UI Framework**: Android XML Views (layouts, fragments, XML resources). **No Jetpack Compose** — not a single `@Composable` function.
- **Design System**: Material Design 3 (Material You) via `com.google.android.material:material`.
- **Architecture**: MVVM with ViewModels, StateFlow, and Repository pattern. Clean separation: UI → domain → data.
- **Build System**: Gradle with **Kotlin DSL** (`build.gradle.kts` only, no Groovy). Version catalog via `gradle/libs.versions.toml`.
- **Dependency Injection**: Hilt (`@HiltAndroidApp`, `@AndroidEntryPoint`, `@HiltWorker`).
- **Networking**: Retrofit + OkHttp for REST API calls.
- **JSON Parsing**: Moshi (with `moshi-kotlin-codegen` via KSP).
- **Build Note**: AGP 9.x has built-in Kotlin support — no separate `kotlin-android` plugin needed.
- **Data Persistence**: DataStore (Preferences) for user preferences, home address, alarm config, cached route data, and signed-in user info.
- **minSdk**: 34 (Android 14). No support below Android 14.
- **compileSdk / targetSdk**: Latest stable API level.

## Key APIs & Services
- **Authentication**: None (Google Sign-In was removed to allow direct app access).
- **Google Places SDK**: Autocomplete for home address input.
- **Google Maps SDK**: For optional map display.
- **Google Routes API v2 (REST)**: For computing transit routes (bus-preferred) via `RoutesApiService.computeRoutes()`. One call per alarm trigger.
- **FusedLocationProviderClient**: For GPS location. Falls back to fresh location update if last known is stale (> 5 min).

## API Cost Awareness
- Operates on Google Maps Platform **free tier only**.
- Cache the last fetched route locally (with timestamp).
- **One API call per alarm trigger**. No polling, no retry loops, no API calls on app launch.
- On failure, allow ONE manual retry via notification action.

## Background Work & Notifications
- **AlarmManager** (`setExactAndAllowWhileIdle()`) for daily alarm. Requires `SCHEDULE_EXACT_ALARM` permission.
- **WorkManager** (expedited `OneTimeWorkRequest`) enqueued by `AlarmReceiver` to fetch route and show notification.
- **NotificationCompat**: Two display modes (user-configurable):
  - **Brief**: Bus number(s) + departure time(s) only.
  - **Detailed (default)**: `BigTextStyle` with full step-by-step route.
- Tap action opens Google Maps with transit directions.
- Error notifications include a "Retry" action button (one retry only).
- `BOOT_COMPLETED` receiver to reschedule alarms after reboot.

## Alarm Modes
- **Recurring (daily)**: Re-schedules after each fire.
- **One-shot (default)**: Fires once, then disables itself.

## Screens
1. **Home Screen**: Greeting, home address input (Places Autocomplete), alarm time picker, alarm enable/disable toggle, recurring/one-shot toggle, next notification status.
3. **Settings Screen**: Notification display mode (brief/detailed), account info + sign-out, about section with Google Maps attribution.
4. **Debug Screen** (dev-only, accessible from Settings): Diagnostic tools for testing alarm/notification flow. Not in original spec — consider hiding in release builds.

## Error Handling Policy
- **Never fail silently.** Every failure must produce user-visible feedback (notification, snackbar, dialog).
- Handle: no internet, API errors, location unavailable, no transit routes found, sign-in token expired, permissions revoked.

## Key Constraints (Do NOT Violate)
- No Jetpack Compose. XML Views only.
- No Groovy build files. Kotlin DSL only.
- No hardcoded API keys. Use `local.properties` + `BuildConfig`.
- No deprecated APIs (`AsyncTask`, legacy support libraries, etc.).
- No unnecessary API calls. Cache results. No polling.
- minSdk 34 — no backward compatibility below Android 14.

## Testing Conventions
- Use Android's `returnDefaultValues = true` for unit tests needing framework stubs.
- Mockito's `mock-maker-subclass` has been explicitly configured in `src/test/resources/mockito-extensions`.
- Core mock target classes (`UserPreferencesDataStore`, `UserPreferencesRepository`, `AlarmScheduler`, `DirectionsRepository`, `NotificationHelper`, `LocationHelper`) are `open` to bypass Mockito `mock-maker-inline` crashes on newer JVM/JDK versions (Java 25).

## Historical Decisions
- Added `ACCESS_BACKGROUND_LOCATION` permission and improved `LocationHelper` retry/fallback for reliable background notification delivery.
- Designed adaptive icon for the app.
- Fixed Material Dialog cancel button requiring two clicks to dismiss.
- Chose Google Routes API v2 over legacy Directions API — reflected in naming: `RoutesApiService`, `RoutesRequest`, `RoutesResponse`.
- Added Debug screen (not in original spec) for development/testing convenience.
- Removed Google Sign-In entirely, allowing users to enter directly to the dashboard, as it was failing on testing devices.
